Implementation Specs

üîí Body Scroll Lock

When results overlay is active, add overflow: hidden to body. This prevents the page beneath from scrolling.

üìê Flex Column Layout

Results overlay uses display: flex; flex-direction: column with header/tabs as flex-shrink: 0 and content as flex: 1; overflow-y: auto.

üéØ Overscroll Behavior

Add overscroll-behavior-y: contain to prevent "rubber-band" scroll from propagating to parent.

üì± Touch Scrolling

Use -webkit-overflow-scrolling: touch for smooth momentum scrolling on iOS devices.

üìç Sticky Header/Tabs

Header and tabs use position: sticky within the flex container to stay visible while cards scroll.

üé® Scroll Indicator

Optional thin progress bar at bottom shows scroll position. Updates via JS scroll event listener.

Key CSS Changes

/* Lock body scroll when results active */
body.locator-results-active {
  overflow: hidden;
  position: fixed;
  width: 100%;
}

/* Results overlay as flex container */
.results-overlay.active {
  display: flex;
  flex-direction: column;
}

/* Fixed header and tabs */
.results-header,
.results-tabs {
  flex-shrink: 0;
}

/* Scrollable content area */
.results-grid {
  flex: 1;
  overflow-y: auto;
  -webkit-overflow-scrolling: touch;
  overscroll-behavior-y: contain;
}
